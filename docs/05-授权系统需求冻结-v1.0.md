# xianyuxian 授权系统需求冻结 v1.0

> 冻结日期：2026-07-11  
> 范围：个人非商业授权中心、Windows 客户端授权接入和个人子管理员页面。

## 1. 已冻结参数

| 项目 | 决定 |
|---|---|
| 用途 | 个人、非商业管理，不提供销售、支付、代理商或财务功能 |
| 云端环境 | Debian VPS，Docker Compose 部署 |
| 访问入口 | 管理专用二级域名，全站 HTTPS |
| 授权类型 | 永久、按天数到期、按自然月到期 |
| 设备数量 | 每个授权码绑定 1 台设备 |
| 闲鱼账号数 | 默认不限制，所有者可按授权设置上限 |
| 子管理员额度 | 每个子管理员最多维护 100 个有效授权码 |
| 首次激活 | 必须联网验证授权码并绑定设备 |
| 日常启动 | 优先联网校验，校验通过后才启动自动任务 |
| 运行心跳 | 每 15 分钟一次 |
| 远程吊销 | 下一次心跳或启动时生效，目标不超过 15 分钟 |
| 离线宽限 | 72 小时，仅服务器不可达时启用 |
| 明确拒绝 | 过期、吊销、封禁、设备不匹配、版本禁用时不得离线放行 |
| 到期行为 | 停止自动任务，保留只读、更新、导出和重新激活 |
| 自助解绑 | 每 30 天最多 2 次，后台人工操作必须审计 |
| 客户端 | Windows 10/11 x64，Python + React + WebView2 |
| 安装方式 | 每用户安装，正常使用无需管理员权限 |
| 后台运行 | 关闭窗口默认最小化到托盘，默认不开机自启 |
| 更新 | 自动检查、提示安装；最低安全版本可强制升级 |
| 数据边界 | 闲鱼 Cookie、卡密、订单、聊天、AI Key 只存本机并加密 |

## 2. 云端角色

### 所有者

- 管理所有授权码、激活设备、子管理员、版本策略和审计记录。
- 设置单个授权的有效期、闲鱼账号上限和状态。
- 发布版本、设置最低版本、禁止版本和强制升级。
- 吊销授权或单台设备，强制注销子管理员会话。

### 子管理员

- 仅管理自己创建或所有者分配的授权码和设备。
- 默认最多维护 100 个有效授权码。
- 可按授权权限生成、续期、停用、吊销和解绑。
- 不能查看其他子管理员数据，不能修改版本、全局策略、所有者和签名密钥。

“有效授权码”指未过期且未吊销/删除的授权。停用但未到期的授权仍占额度，以防通过停用绕过数量限制。

## 3. 时间规则

- 服务端使用 UTC 存储时间，管理后台按 `Asia/Shanghai` 显示和输入。
- 按天授权：生效时刻加 N×24 小时。
- 按自然月授权：使用日历月加法；目标月份不存在对应日期时取该月最后一天。
- 未激活授权默认从首次激活开始计时，也可由管理员指定固定生效/到期时间。
- 未到期续期从原到期时间累加；已到期续期从续期操作时间重新计算。
- 永久授权没有固定到期时间，但仍可远程吊销，并受版本策略约束。

## 4. 域名与路径

假设实际域名为 `license.<主域名>`：

- `/admin/`：所有者和子管理员页面。
- `/admin/v1/`：管理 API。
- `/v1/`：Windows 客户端授权 API。
- `/downloads/`：可选安装包下载路径。

实际域名只通过部署环境变量和客户端生产构建配置注入，不写死 VPS IP。

## 5. 实现阶段可填写但不改变需求的参数

- VPS IP、CPU、内存、磁盘、SSH 用户和地域。
- 实际主域名/二级域名及 DNS 服务商。
- 子管理员总人数和每人最长可签发时间。
- 安装包存储位置和稳定/测试渠道下载地址。
- 产品名称、Logo、发布者和版本号起点。

这些属于部署或品牌配置，不阻塞授权系统的代码结构设计。
