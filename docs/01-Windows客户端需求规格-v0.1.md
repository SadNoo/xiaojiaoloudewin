# xianyuxian Windows 客户端需求规格 v0.1

> 状态：部分需求已确认，剩余参数待评审  
> 已确认日期：2026-07-11

## 0. 已确认的产品边界

- 本项目为个人、非商业用途，不提供授权码销售、支付或代理商结算。
- 必须建设独立在线授权中心，可部署在 Cloud 或自有 VPS。
- 云端必须提供授权码、设备绑定、远程吊销、离线宽限和客户端版本控制。
- 云端必须提供所有者管理页面以及个人子管理员的授权管理页面。
- 授权类型同时支持永久授权、按天数到期和按自然月到期。
- 每个授权码默认只绑定 1 台设备；默认不限制闲鱼账号数量，所有者可调整单个授权的账号上限。
- 每个个人子管理员默认最多维护 100 个处于有效状态的授权码。
- 授权中心部署在 Debian VPS，通过管理专用二级域名提供 HTTPS 服务。
- Windows 客户端首次激活必须联网；日常启动必须优先联网校验授权，只有已经成功激活且授权服务器临时不可达时，才允许使用服务端签名的离线宽限票据。
- 闲鱼 Cookie、订单、聊天、卡券和 AI Key 默认仅存本机，授权中心不接收这些业务明文数据。

## 1. 产品目标

把现有闲鱼超级管家改造成可安装、可升级、可诊断的 Windows 桌面客户端，并通过自有在线授权 API 管理使用资格。客户端应尽量在本机完成闲鱼账号、订单、卡券、回复和 AI 配置处理，授权服务只管理许可证、设备和版本策略。

### 1.1 首版目标用户

- 使用 Windows 10/11 64 位的单机闲鱼卖家或运营人员。
- 需要同时管理一个或多个闲鱼账号。
- 需要自动回复、AI 议价、订单同步和虚拟卡券自动发货。
- 通过所有者或个人子管理员分配的授权码获得客户端使用权限。

### 1.2 首版不建议包含

- 云端托管闲鱼 Cookie 或代运行闲鱼账号。
- 多电脑实时同步卡密、Cookie、聊天和订单。
- 手机客户端、macOS 客户端。
- 绕过平台风控、验证码或账号安全机制的承诺。
- 客户端内的授权码销售、退款和财务结算后台。

## 2. 核心用户流程

### 2.1 首次使用

1. 用户安装并启动客户端。
2. 客户端展示隐私和风险提示，用户确认。
3. 用户输入授权码。
4. 客户端生成隐私友好的设备指纹，调用授权 API 激活。
5. 授权成功后，客户端在本机创建加密凭证和数据目录。
6. 用户创建本地管理员密码或使用 Windows 当前用户保护应用。
7. 用户通过扫码、账号登录或 Cookie 添加闲鱼账号。
8. 用户配置回复、AI、卡券和发货规则。
9. 用户启动账号任务，并在仪表盘查看运行状态。

### 2.2 日常启动

1. 客户端读取本机加密授权凭证。
2. 客户端调用授权中心完成本次在线校验；若服务端不可达，才检查最近一次在线校验后签发的离线票据。
3. 启动本地业务内核和账号任务。
4. UI 展示授权到期日、在线状态、任务状态和异常告警。

在线验证成功后，客户端运行期间每 15 分钟发送一次授权心跳；远程吊销应在下一次心跳或下一次启动时生效。离线宽限不是跳过首次联网激活，也不能覆盖“服务端明确返回已吊销、已过期、设备不匹配或版本被禁用”的结果。

### 2.3 换机或解绑

1. 用户在旧设备主动停用并释放设备名额；或在授权管理后台解绑。
2. 新设备使用同一授权码重新激活。
3. 本地业务数据默认不自动迁移；用户可导出加密备份后导入。

## 3. 功能需求

### 3.1 授权与设备管理（P0）

- 支持授权码激活、状态查询、静默刷新、主动停用和重新激活。
- 支持永久、按天数、按自然月三种有效期，支持设备数上限和功能权益；个人版不包含销售套餐和支付能力。
- 有效期计算统一使用服务端时间和 `Asia/Shanghai` 时区，客户端时间不作为授权到期的可信依据。
- 按天授权以激活或管理员指定的生效时刻加 N×24 小时计算；按自然月授权使用日历月加法。
- 自然月遇到目标月份不存在的日期时取该月最后一天，例如 1 月 31 日加 1 个自然月到 2 月最后一天同一时刻。
- 永久授权仍可被所有者或有权限的子管理员主动吊销，也受禁止版本和最低版本策略约束。
- 授权状态至少包括：未激活、有效、即将到期、已过期、已停用、已封禁、设备超限、服务器不可达、客户端版本过低。
- 支持服务端吊销授权或单台设备。
- 支持 72 小时离线宽限期，宽限只在授权服务器无法连接时启用。
- 设备指纹应可解释并容忍普通硬件/系统小变更，不读取无关个人文件。
- 本机只保存授权码的不可逆摘要或受 DPAPI 保护的必要值，不明文写日志。
- 授权检查失败时不得破坏或删除用户本地数据。
- 到期后停止自动化任务，允许进入只读界面、更新、导出数据和重新激活。
- 客户端允许主动解绑当前设备，每 30 天最多自助解绑 2 次；后台人工解绑不受此次数限制但必须审计。

### 3.2 本地账号与应用访问（P0）

- 首次初始化本地管理员。
- 支持锁屏、退出登录、修改密码和会话超时。
- 桌面版默认单用户；保留多用户数据结构但首版不开放复杂 RBAC。
- 可选“随 Windows 启动”“最小化到托盘”“关闭窗口时继续运行”。
- 默认不随 Windows 启动；关闭主窗口默认最小化到托盘，只有托盘“退出”才停止全部任务。

### 3.3 闲鱼账号管理（P0）

- 多账号添加、删除、启停、备注和运行状态查看。
- 支持扫码登录；密码登录和手动 Cookie 作为可配置能力。
- 展示 Cookie 是否有效、上次刷新、最近心跳和最近错误。
- 遇到滑块、人脸或其他验证时停止自动操作并提示人工完成。
- 每账号可配置任务启停、暂停时长、自动确认和 AI 设置。

### 3.4 自动回复（P0）

- 关键词回复，支持全局和商品专属规则。
- 默认回复、单会话一次、文本和图片回复。
- 规则优先级、启停、命中测试和冲突提示。
- AI 回复使用用户提供的模型 API Key；支持 OpenAI 兼容接口。
- AI 议价有折扣比例、金额、轮数等硬限制，AI 不得越过规则直接成交。
- 失败时可降级为关键词或默认回复，并记录脱敏原因。

### 3.5 商品、订单和卡券（P0）

- 商品同步、搜索、详情、专属回复和多规格配置。
- 订单分页、筛选、详情、单条/批量同步和状态展示。
- 卡券分组、批量导入、库存、已用/未用状态和导出。
- 发货规则按商品/关键词/规格匹配，并支持启停、优先级和测试。
- 自动发货需要幂等控制，避免同一订单重复消耗卡券。
- 手动补发提供“仅修改状态”和“完整发货”，必须二次确认。
- 关键操作写本地审计日志，但不得记录完整卡密。

### 3.6 运行监控与诊断（P0）

- 仪表盘展示各账号在线、监听、授权、订单和库存状态。
- 托盘图标展示正常、警告、离线三类状态。
- 提供可读错误信息、重试入口和诊断包导出。
- 诊断包默认脱敏，用户预览并确认后才能用于客服排障。
- 单个账号任务崩溃不应导致全部账号或桌面 UI 退出。

### 3.7 配置、备份与恢复（P1）

- 所有业务配置从 UI 管理，避免要求用户编辑 YAML。
- 支持加密备份导出和导入，包含规则、卡券、商品映射和可选账号信息。
- 导入前校验版本和结构，失败时保持原数据库不变。
- 支持恢复到导入前备份。
- 用户可选择清空业务数据、授权信息或全部本地数据。

### 3.8 自动更新（P1）

- 客户端启动或用户手动检查更新。
- 支持稳定版渠道；测试版渠道可在后续开放。
- 服务端可设置最低支持版本和强制升级原因。
- 更新包必须签名并校验完整性，下载失败不影响现有版本启动。
- 数据库升级前自动备份，升级失败可回滚。
- 版本策略至少支持：最新版本、最低可用版本、强制升级、禁止版本、维护公告和稳定/测试渠道。
- 版本发布只能由所有者执行；子管理员默认只读版本信息。

### 3.9 个人授权管理后台（独立云端项目，P0）

- 所有者和个人子管理员登录，支持修改密码、会话失效和操作审计。
- 创建、批量生成、导出、禁用和续期授权码。
- 配置有效期、设备上限、功能权益、版本渠道、备注和归属子管理员。
- 查询激活设备、最近在线、客户端版本和授权事件。
- 设备解绑、授权吊销、异常频率限制。
- 所有者可创建、停用、重置个人子管理员，并为其分配授权码额度、可管理范围和权限。
- 每个子管理员默认最多拥有 100 个有效授权码；已过期、已吊销和已删除的授权码不占有效额度，停用但尚未到期的授权码仍占额度。
- 子管理员只能查看和管理分配给自己或由自己创建的授权码和设备，不得查看其他子管理员数据。
- 子管理员不能修改系统签名密钥、所有者账号、全局版本策略、服务配置或审计日志。
- 所有生成、续期、解绑、吊销、版本发布和权限变更操作必须记录操作者、时间、来源 IP、对象和结果。
- 不展示或收集用户的闲鱼 Cookie、聊天、卡密和订单明文。

## 4. 在线授权 API 需求

### 4.1 建议端点

| 方法 | 路径 | 用途 |
|---|---|---|
| `POST` | `/v1/licenses/activate` | 授权码 + 设备信息激活 |
| `POST` | `/v1/licenses/refresh` | 使用刷新凭证换短期访问凭证 |
| `GET` | `/v1/licenses/status` | 获取授权、权益和最低版本策略 |
| `POST` | `/v1/licenses/deactivate` | 主动释放当前设备 |
| `POST` | `/v1/licenses/heartbeat` | 运行期间在线复核授权和吊销状态 |
| `GET` | `/v1/releases/latest` | 查询适用渠道的最新版本和包信息 |
| `GET` | `/v1/releases/manifest/{version}` | 获取签名版本清单、下载地址和校验值 |

管理后台 API 使用独立的 `/admin/v1/` 前缀，至少包含管理员登录、子管理员、授权码、激活设备、吊销、版本和审计日志资源。管理 API 与客户端授权 API 必须使用不同的认证策略和速率限制。

云端通过一个管理专用二级域名提供服务，例如 `license.<主域名>`。管理页面使用 `/admin/`，后台 API 使用 `/admin/v1/`，客户端授权 API 使用 `/v1/`；反向代理统一终止 HTTPS。实际域名在部署前写入环境配置，不硬编码到源码。

### 4.2 激活请求建议字段

- `license_code`：用户输入的授权码，仅通过 HTTPS 发送。
- `device_id`：基于设备材料生成的稳定派生标识，不直接上传原始硬件序列号。
- `device_name`：用户可识别的设备名，可编辑。
- `app_version`、`channel`、`os_version`、`architecture`。
- `request_id`、客户端时间和随机数，用于幂等与重放防护。

### 4.3 激活响应建议字段

- `activation_id`、`license_status`、`expires_at`。
- `access_token`：短期使用，建议 15～60 分钟。
- `refresh_token`：设备绑定，服务端可单独吊销并轮换。
- `offline_ticket`：服务端签名、含截止时间和权益的离线声明。
- `entitlements`：账号数、功能开关、版本渠道等。
- `server_time`、`minimum_version`、`update_policy`。

### 4.4 在线校验状态机

1. 未激活客户端没有离线资格，必须联网提交授权码和设备标识。
2. 已激活客户端每次启动先调用刷新或状态接口，成功后才启动闲鱼自动化任务。
3. 服务端明确返回已过期、已吊销、已封禁、设备不匹配或版本不允许时，客户端立即拒绝启动自动任务，不得回退到离线票据。
4. 只有 DNS、连接、TLS、超时或服务端 5xx 等“服务器不可达”错误，才允许校验离线票据。
5. 离线票据必须未过期、设备匹配、版本满足策略且签名有效；否则进入锁定或只读模式。
6. 离线期间继续按退避策略尝试恢复联网，恢复后立即应用最新吊销和版本策略。
7. 客户端运行期间按心跳周期复核；连续失败进入宽限倒计时，收到明确吊销则停止自动化任务并保存本地数据。

### 4.5 安全约束

- 全部 API 使用 HTTPS，服务端证书正常校验；是否增加证书固定待威胁模型后决定。
- 服务端数据库只保存授权码的慢哈希或 HMAC 索引，不保存可直接使用的明文授权码。
- 访问凭证短期有效，刷新凭证轮换，旧凭证重用触发风控。
- 离线票据使用非对称签名，客户端只内置公钥，不能内置可签发许可证的私钥或共享密钥。
- 请求和授权事件有幂等键、速率限制、审计日志和异常告警。
- 客户端时间异常时以服务器时间锚点判断，不因简单改系统时间延长授权。
- 任何客户端防篡改只能提高成本，最终授权判断和吊销能力必须在服务端。
- 云端签名私钥不得放入客户端、前端代码、Git 仓库或普通数据库字段；建议由服务端密钥文件或秘密管理服务加载并限制文件权限。
- 所有者和子管理员账号建议支持 TOTP 双因素认证；所有者必须能够强制注销子管理员全部会话。

## 5. 非功能需求

### 5.1 支持环境

- Windows 10 22H2 与 Windows 11，x64；是否支持 ARM64 待确认。
- 首版不支持 Windows Server 和 Windows ARM64。
- 依赖 Microsoft Edge WebView2 Runtime；安装程序应检测并给出明确引导。
- 建议最低 4 核 CPU、8 GB 内存；多账号和浏览器自动化按账号数增加资源。

### 5.2 性能与稳定性

- 冷启动目标：普通电脑 10 秒内显示主窗口，后台任务可稍后就绪。
- UI 操作不被浏览器任务阻塞。
- 非浏览器常驻内存目标待基线测试后确定。
- 自动化任务失败采用有上限的指数退避，避免频繁触发风控。
- 进程异常退出后，下次启动可检测未完成订单/发货事务并安全恢复。

### 5.3 安全与隐私

- 敏感字段使用应用层加密，根密钥通过 Windows DPAPI 绑定当前用户或设备。
- 本地数据库和日志放在规范的 `%LOCALAPPDATA%/xianyuxian/` 路径。
- 不在命令行参数、崩溃报告和更新日志中包含凭证。
- 默认关闭遥测；如未来启用，必须明确告知、最小化收集并提供关闭选项。
- 备份默认加密，导出敏感数据要求再次确认。

### 5.4 可维护性

- 授权、桌面壳、业务内核、闲鱼 adapter、数据层和 UI 各自有清晰边界。
- API 使用版本前缀并维护前后端兼容矩阵。
- 数据库采用显式 schema migration。
- P0 业务流程具备单元测试或可重复的集成测试。
- 构建可复现，发布产物带版本、提交号、签名和 SHA-256。

## 6. 建议桌面技术方案

### 方案 A：Python + pywebview/WebView2（建议首版）

- 复用 React 构建产物和 Python 内核，改造量较小。
- 可用 Nuitka 或 PyInstaller 打包，Playwright 浏览器作为受控运行时随安装包部署。
- 需要自行实现单实例、托盘、更新、崩溃处理和本地 API 防护。

### 方案 B：Tauri + React + Python sidecar

- 桌面体验、安装包和更新体系较好。
- Rust/Tauri 与 Python sidecar 的进程通信、签名和打包更复杂。
- 适合第二阶段或团队具备 Rust/Tauri 经验时采用。

### 方案 C：Electron + React + Python sidecar

- 桌面生态成熟，托盘和更新较容易。
- Electron 自带 Chromium，而 Playwright 可能还需独立浏览器，安装体积和内存最大。

首版确定选择方案 A，以最小结构变化验证产品；同时保持业务内核与桌面壳解耦，未来可以替换为 Tauri。

## 7. 验收标准摘要

- 全新 Windows 10/11 环境可通过安装包完成安装、激活和启动。
- 有效授权可进入业务功能；过期授权停止自动任务但允许导出本地数据。
- 首次激活和正常启动完成在线授权校验；仅服务端不可达时可使用有效离线票据。
- 断网在约定宽限期内可用，超过宽限期按策略降级；服务端明确吊销时不得进入宽限。
