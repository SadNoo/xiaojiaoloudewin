# xianyuxian 需求确认清单

> 已确认项使用 `[x]`；请继续确认未完成项。标注“建议”的选项用于形成默认方案。

## A. 商业与授权范围（必须先确认）

- [x] A1. 项目定位为个人、非商业用途，不销售授权码、不提供支付或代理商结算。
- [x] A2. 授权有效期同时支持：`永久`、`按天数到期`、`按自然月到期`。
- [x] A3. 一个授权码只允许绑定 `1 台设备`。
- [x] A4. 首版不按商业套餐区分权益；默认不限制闲鱼账号数量，所有者可在后台对单个授权设置账号上限。
- [x] A5. 本项目一并开发自有授权 API、所有者页面和个人子管理员页面，不接入商业授权平台。
- [x] A6. 授权服务部署在 Debian VPS，通过管理专用二级域名提供 HTTPS 服务；具体域名待部署时填写。

## B. 在线 API 的业务边界（必须确认）

- [x] B1. 在线 API 首版负责授权码、设备绑定、远程吊销、离线票据和版本控制，不提供闲鱼业务云同步。
- [x] B2. 闲鱼 Cookie、卡密、订单、聊天和 AI Key 全部仅保存在本机并加密。
- [x] B3. 需要 Web 授权管理后台，包含所有者和个人子管理员权限。
- [x] B4. 断网宽限期为 `72 小时`，仅授权服务器不可达时启用。
- [x] B5. 授权到期后停止自动任务，但允许只读、更新、重新激活和数据导出。
- [x] B6. 允许客户端主动解绑设备，每 30 天最多自助解绑 2 次；所有者或有权限的子管理员可人工处理。
- [x] B7. 不做邀请码销售、代理商、在线支付、退款或财务结算。
- [x] B8. 客户端首次激活必须联网；每次启动优先联网验证；仅服务不可达时允许进入离线宽限。
- [x] B9. 客户端运行时授权心跳周期为 `15 分钟`；远程吊销目标在下一次心跳内生效。
- [x] B10. 每个子管理员默认最多维护 `100` 个有效授权码；已过期和已吊销授权不占额度。

## C. Windows 客户端形态（必须确认）

- [x] C1. 首版支持 Windows 10/11 x64，不要求 Windows Server 或 ARM64。
- [x] C2. 首版采用 `Python + React + WebView2`，保持业务内核和桌面壳可替换。
- [x] C3. 采用每用户安装，正常安装和运行无需管理员权限。
- [x] C4. 个人内测阶段不强制购买代码签名证书，但发布包必须有 SHA-256 和项目自身签名；未来公开分发再接入 Windows 代码签名。
- [x] C5. 关闭主窗口默认最小化到托盘继续运行，托盘菜单提供“退出并停止全部任务”。
- [x] C6. 默认不随 Windows 启动，由用户主动开启。
- [x] C7. 自动检查更新，下载后提示安装；低于安全最低版本时允许强制升级。
- [ ] C8. 产品中文名、英文名、Logo、主色、公司/个人署名和安装包发布者名称。

## D. 原项目功能取舍（必须确认）

请为每项标记首版是否保留：

- [ ] D1. 多闲鱼账号。
- [ ] D2. 扫码登录。
- [ ] D3. 闲鱼账号密码登录。
- [ ] D4. 手动粘贴 Cookie。
- [ ] D5. 关键词、默认和商品专属回复。
- [ ] D6. AI 回复和 AI 议价。
- [ ] D7. 卡券库存与自动发货。
- [ ] D8. 多规格和延时发货。
- [ ] D9. 订单同步、导入、补发和统计。
- [ ] D10. 商品搜索与商品同步。
- [ ] D11. 通知渠道。
- [ ] D12. 多个本地用户及管理员后台。
- [ ] D13. 数据备份导入导出。
- [ ] D14. 滑块/人脸验证的人工辅助界面。

建议首版保留 D1、D2、D4～D9、D13、D14；D3、D10～D12 根据实际用户价值和维护成本再定。

- [x] F1. 敏感数据默认仅保存在本机并加密。
- [x] F2. 首版默认关闭匿名遥测和崩溃上报，未来如增加则由用户主动开启。
- [x] F3. 排障使用用户主动导出的脱敏诊断包。
