# 授权门禁与激活界面集成报告

> 完成日期：2026-07-13  
> 阶段状态：授权门禁已接入现有启动流程，前端激活页已构建。

## 启动流程变化

原流程是启动程序后立即创建 `CookieManager` 并启动所有启用账号。新流程为：

1. 初始化日志和本地数据库。
2. 仅在 `127.0.0.1` 启动本地 FastAPI/UI。
3. 初始化 Windows 授权模块并执行启动在线验证。
4. 在线有效或离线票据有效时，创建 `CookieManager` 并启动闲鱼账号任务。
5. 未激活、过期、吊销、设备不匹配或需要强制升级时，不创建闲鱼任务，前端显示对应授权页面。
6. 心跳状态从允许变为拒绝时，取消全部账号任务、关闭共享浏览器池并保留本地数据。

只有显式设置 `ALLOW_REMOTE_API=1` 才允许本地业务 API 使用非回环监听地址；Windows 客户端默认不能从局域网访问。

## 本地授权 API

- `GET /api/license/status`：返回适合 UI 展示的状态，不返回刷新令牌或离线票据。
- `POST /api/license/activate`：输入授权码并联网激活当前设备。
- `POST /api/license/retry`：重新执行在线刷新或离线判断。
- `POST /api/license/deactivate`：本地管理员主动解绑当前设备。

未授权时，GET/HEAD/OPTIONS 查询和数据导出保持可用；登录、授权和必要验证码写操作保持可用；其他 POST/PUT/DELETE 业务操作返回 HTTP 423。

## 前端界面

- 启动时先检查授权，再检查本地管理员会话。
- 未激活时显示授权码输入框和当前设备名称。
- 授权配置错误时显示构建配置提示。
- 版本过低时显示最低版本和最新版本。
- 授权服务器不可达但离线票据有效时进入业务界面，并显示离线宽限截止时间。
- 每 30 秒读取一次本地授权状态；心跳吊销、过期或强制升级后自动退出业务界面。
